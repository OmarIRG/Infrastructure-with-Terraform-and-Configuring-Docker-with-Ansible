Omar Islam Ragab ----- Infrastructure Automation using Terraform and Ansible

Project Overview

This project automates the provisioning and configuration of infrastructure using Terraform and Ansible. It leverages the GitOps methodology, ensuring that all infrastructure and configuration code is managed through Git, providing an auditable, version-controlled setup.

Key Components

  • Terraform: Used to provision cloud infrastructure (servers, networking, etc.)
  • Ansible: Used for configuring the provisioned servers (e.g., installing Docker)
  • CI/CD Pipeline: Automates the deployment and configuration process using tools like GitHub Actions, GitLab CI, or Jenkins

GitOps Implementation

The project adheres to the GitOps principles:

  1. All infrastructure and configuration code is stored in a Git repository.
  2. CI/CD pipelines are used to automatically deploy and configure infrastructure.
  3. Any changes to the infrastructure or configuration are managed through Git pull requests and commit histories.

Instructions

Follow these steps to deploy the infrastructure and configure the servers:

  1. Clone the repository.
  2. Run terraform apply to provision the infrastructure.
  3. Use the generated Ansible inventory file to run the playbook on the bastion host.
  4. The pipeline will automatically handle further deployments.
